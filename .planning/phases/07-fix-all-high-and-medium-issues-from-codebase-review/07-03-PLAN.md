**Issue 6 — Engine leaked on release() if reset_workspace() raises (pool/manager.py:115-123):**
Rewrite `release()` to use try/finally:
```python
async def release(self, engine: MatlabEngineWrapper) -> None:
    logger.info("Releasing engine %s — resetting workspace", engine.engine_id)
    loop = asyncio.get_running_loop()
    try:
        await loop.run_in_executor(None, engine.reset_workspace)
    except Exception as exc:
        logger.warning(
            "[%s] reset_workspace failed on release: %s — returning engine to pool",
            engine.engine_id, exc,
        )
        engine._needs_replacement = True
    finally:
        engine.mark_idle()
        await self._available.put(engine)
```
Note: `engine._needs_replacement = True` is a new flag. It will be checked during health checks.

**Issue 7 — engine_start_timeout never enforced (pool/manager.py:54-59):**
Rewrite `_start_engine_async()` to use `asyncio.wait_for()`:
```python
async def _start_engine_async(self) -> MatlabEngineWrapper:
    loop = asyncio.get_running_loop()
    engine = self._make_engine()
    self._all_engines.append(engine)
    timeout = float(self._pool_config.engine_start_timeout)
    try:
        await asyncio.wait_for(
            loop.run_in_executor(None, engine.start),
            timeout=timeout,
        )
    except asyncio.TimeoutError:
        logger.error("[%s] Engine startup timed out after %.0fs", engine.engine_id, timeout)
        try:
            self._all_engines.remove(engine)
        except ValueError:
            pass
        raise RuntimeError(
            f"Engine {engine.engine_id} failed to start within {timeout:.0f}s"
        )
    return engine
```

**Issue 8 — Race in acquire() after _scale_lock (pool/manager.py:96-105):**
After the `_scale_lock` block, before the blocking `await self._available.get()`, add a `get_nowait()` re-poll:
```python
# Re-poll: an engine may have been returned while we held the scale lock
try:
    engine = self._available.get_nowait()
    engine.mark_busy()
    return engine
except asyncio.QueueEmpty:
    pass
# Now block until an engine is available
engine = await self._available.get()
engine.mark_busy()
return engine
```

**Issue 23 — get_status() busy count (pool/manager.py:218-231):**
Replace the queue-arithmetic approach with direct state counting:
```python
def get_status(self) -> Dict[str, int]:
    total = len(self._all_engines)
    busy = sum(1 for e in self._all_engines if e.state == EngineState.BUSY)
    available = self._available.qsize()
    return {
        "total_engines": total,
        "available_engines": available,
        "busy_engines": busy,
    }
```

**Issue 24 — Executor accesses engine._engine.workspace directly (executor.py:178,184,275):**

Add two new methods to `MatlabEngineWrapper`:

```python
def set_workspace_var(self, name: str, value: Any) -> None:
    """Set a variable in the MATLAB workspace via the engine API."""
    if self._engine is None:
        raise RuntimeError("Engine not started")
    self._engine.workspace[name] = value

def get_workspace_vars(self) -> Any:
    """Return the workspace proxy object for reading variables."""
    if self._engine is None:
        raise RuntimeError("Engine not started")
    return self._engine.workspace
```

Also add a `_needs_replacement: bool = False` attribute in `__init__()` (used by the release fix in Task 1):
```python
self._needs_replacement: bool = False
```
