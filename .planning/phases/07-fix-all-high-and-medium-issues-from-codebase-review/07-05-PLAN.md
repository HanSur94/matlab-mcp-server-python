**Issue 34 — Remove _config_dir dead attribute (config.py:181):**
Find `_config_dir: Optional[Path] = None` in `AppConfig`. Remove it entirely. It is a Pydantic v2 private attribute that is never read — the value is passed directly to `resolve_paths()`.

**Issue 26 — Remove proactive_warmup_threshold dead config (config.py:43):**
Find `proactive_warmup_threshold` in `PoolConfig`. Remove the field. Search the codebase for any references to this field and remove them (there should be none since it is dead config). If there are references, leave them as TODOs.

**Issue 18 — Make inspect_mode a proper field (config.py + server.py):**
Add to `AppConfig`:
```python
inspect_mode: bool = Field(default=False, exclude=True)
```
This replaces the dynamic `config._inspect_mode = True` assignment in server.py. The `exclude=True` prevents it from appearing in serialization.

**Issue 19 — Configurable CORS origins (config.py + server.py):**
Add to `ServerConfig`:
```python
cors_origins: list[str] = Field(default_factory=lambda: ["*"])
```
This makes CORS origins configurable via config.yaml under `server.cors_origins`.

**Issue 33 — YAML parse error handling (config.py:287-306):**
In `load_config()`, wrap the `yaml.safe_load(fh)` call:
```python
try:
    loaded = yaml.safe_load(fh) or {}
except yaml.YAMLError as exc:
    raise ValueError(f"Failed to parse config file {path}: {exc}") from exc
```

**Issue 16 — asyncio.get_event_loop() deprecated (server.py:357-366):**
Find all occurrences of `asyncio.get_event_loop()` in server.py. Replace with `asyncio.get_running_loop()`. There should be ~2 occurrences in the lifespan finally block. Since the code is inside an async context (the lifespan generator), `get_running_loop()` is correct.

**Issue 17 — _get_session_id swallows exceptions (server.py:103-128):**
Find `_get_session_id` function. Where it catches exceptions with a bare `except` or `except Exception`, change to log the exception:
```python
except Exception as exc:
    logger.warning("Failed to extract session_id from context: %s", exc)
    return _DEFAULT_SESSION_ID
```
Instead of silently returning the default.

**Issue 18 continued — Replace dynamic _inspect_mode assignment (server.py:849-851):**
Find `config._inspect_mode = True` (or similar). Replace with:
```python
config.inspect_mode = True
```
Remove any `# type: ignore[attr-defined]` comment on that line. Since `inspect_mode` is now a proper field (from Task 1), this is a normal attribute assignment. Note: If AppConfig is frozen (model_config has frozen=True), the assignment must happen before the freeze or use `model_copy(update={"inspect_mode": True})`. Check the model_config and handle accordingly.

**Issue 19 continued — Wire CORS origins from config (server.py:862-866):**
Find where `CORSMiddleware` is configured with `allow_origins=["*"]`. Replace with:
```python
allow_origins=config.server.cors_origins
```

**Issue 25 — Double-timeout pattern (executor.py:110-114):**
Find the double-timeout code where `future.result(timeout=sync_timeout)` is wrapped in `asyncio.wait_for(..., timeout=sync_timeout + 1)`. Add a clear comment explaining why both exist:
```python
# Inner timeout: MATLAB Engine API honors future.result(timeout=N).
# Outer timeout (+1s safety): asyncio fallback if Engine API ignores timeout.
# The inner fires first under normal conditions; outer is a safety net only.
```
Alternatively, if the research recommendation to remove the outer is preferred, remove the `asyncio.wait_for` wrapper and just use:
```python
result = await loop.run_in_executor(None, lambda: future.result(timeout=sync_timeout))
```
with a comment explaining that asyncio cancellation does not interrupt thread executors.
