  <action>
    Add a `test-macos` job to `.github/workflows/ci.yml` after the `test-windows` job and before the `docker` job. The job should:

    1. Depend on `lint` (same as other test jobs): `needs: lint`
    2. Use `macos-latest` runner
    3. Matrix strategy with `fail-fast: false` and Python versions `["3.10", "3.12"]`
    4. Steps:
       - `actions/checkout@v4`
       - `actions/setup-python@v5` with `python-version: ${{ matrix.python-version }}`
       - `pip install -e ".[dev,monitoring]"` (same as Linux test job)
       - `pytest tests/ -v -k "not matlab" -W ignore::pytest.PytestUnraisableExceptionWarning`

    The exact YAML block to insert:

    ```yaml
    test-macos:
      needs: lint
      runs-on: macos-latest
      strategy:
        fail-fast: false
        matrix:
          python-version: ["3.10", "3.12"]
      steps:
        - uses: actions/checkout@v4
        - uses: actions/setup-python@v5
          with:
            python-version: ${{ matrix.python-version }}
        - run: pip install -e ".[dev,monitoring]"
        - run: pytest tests/ -v -k "not matlab" -W ignore::pytest.PytestUnraisableExceptionWarning
    ```

    Notes:
    - Do NOT use `install.bat` (that is Windows-only). Use direct `pip install` like the Linux job.
    - The `-k "not matlab"` filter excludes tests requiring a MATLAB engine (not available in CI).
    - The `-W ignore::pytest.PytestUnraisableExceptionWarning` suppresses async cleanup warnings (matches Windows job pattern).
    - No coverage upload for macOS (only Linux uploads to Codecov).
  </action>
