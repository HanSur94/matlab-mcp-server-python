**Title**: "Connecting AI Agents to MATLAB MCP Server"

**Overview** (brief):
- The MATLAB MCP server speaks the Model Context Protocol. Any MCP-compatible agent can connect.
- Two transport modes: **stdio** (single agent, local, no auth) and **streamable HTTP** (multi-agent, remote-capable, auth required).
- This guide shows exact configs for Claude Code, Codex CLI, and Cursor.

**Prerequisites** (short checklist):
- MATLAB MCP server installed and working (link to `windows-deployment.md` for Windows setup)
- For HTTP transport: server running with `--transport streamablehttp` and `MATLAB_MCP_AUTH_TOKEN` set
- Server URL: `http://127.0.0.1:8765/mcp` (default)

**Section 1: Claude Code**

*Option A -- stdio (recommended for local single-user):*
Show the `.claude/claude_desktop_config.json` (or `.mcp.json` for Claude Code CLI) entry:
```json
{
  "mcpServers": {
    "matlab": {
      "command": "matlab-mcp",
      "args": ["--config", "path/to/config.yaml"]
    }
  }
}
```
Note: stdio requires no auth token. The agent spawns the server process directly.

*Option B -- streamable HTTP:*
```json
{
  "mcpServers": {
    "matlab": {
      "type": "streamable-http",
      "url": "http://127.0.0.1:8765/mcp",
      "headers": {
        "Authorization": "Bearer ${MATLAB_MCP_AUTH_TOKEN}"
      }
    }
  }
}
```
Note: Set `MATLAB_MCP_AUTH_TOKEN` in your shell before launching Claude Code. The server must be running separately.

**Section 2: Codex CLI (OpenAI)**

Codex CLI uses an MCP config file. Show the exact config:

*Option A -- stdio:*
```json

  "mcpServers": {
```

*Option B -- streamable HTTP:*

Note: Codex CLI connects to `/mcp` via streamable HTTP. This is the transport that resolved the original Codex CLI connectivity issue (SSE was incompatible).

**Section 3: Cursor**

Cursor supports MCP servers via Settings > MCP. Show:

*Option A -- stdio (via settings UI):*
- Open Cursor Settings > MCP
- Add server: command = `matlab-mcp`, args = `--config path/to/config.yaml`

*Option B -- streamable HTTP (via `.cursor/mcp.json`):*
```json
{
  "mcpServers": {
    "matlab": {
      "type": "streamable-http",
      "url": "http://127.0.0.1:8765/mcp",
      "headers": {
        "Authorization": "Bearer ${MATLAB_MCP_AUTH_TOKEN}"
      }
    }
  }
}
```

**Section 4: Other MCP-Compatible Agents**

Generic instructions:
- stdio: spawn `matlab-mcp` with optional `--config` flag
- HTTP: connect to `http://<host>:<port>/mcp` with `Authorization: Bearer <token>` header
- The server implements MCP 1.26.0 protocol

**Section 5: Verifying the Connection**

After configuring your agent:
1. Ask the agent to "list available MATLAB toolboxes" -- this calls `list_toolboxes` tool
2. Ask it to "run `disp('Hello from MATLAB')`" -- this calls `execute_code` tool
3. Expected: agent returns MATLAB output with no errors

**Section 6: Troubleshooting**

- "Connection refused": Server not running or wrong port. Check `matlab-mcp` is running with `--transport streamablehttp`.
- "401 Unauthorized": Token mismatch. Regenerate with `matlab-mcp --generate-token` and update both server env and agent config.
- "Agent doesn't see MATLAB tools": MCP handshake failed. Check agent logs for connection errors. Try `--inspect` mode for debugging.
- "SSE transport errors with Codex CLI": Codex CLI requires streamable HTTP, not SSE. Set transport to `streamablehttp`.

Use consistent Markdown formatting. Each agent section should be self-contained so a reader can jump directly to their agent.
