- Option A: `pip install matlab-mcp-python` (from PyPI)

**Configuration section**:
- Create a minimal `config.yaml` for streamable HTTP (the recommended transport for agents):
  ```yaml
  server:
    transport: "streamablehttp"

  ```
- Explain `host: "127.0.0.1"` is the default and avoids Windows Firewall UAC prompts. Add a callout box: "To accept connections from other machines, set `host: \"0.0.0.0\"` -- this requires an admin-created Windows Firewall rule for port 8765."

**Authentication section**:
- Generate a token: `matlab-mcp --generate-token`
- Set it: `set MATLAB_MCP_AUTH_TOKEN=<token>` (cmd) or `$env:MATLAB_MCP_AUTH_TOKEN="<token>"` (PowerShell)
- Explain: token is read from env var only, never stored in config.yaml.
- For stdio transport (single user, local): auth is not required.

**Starting the server section**:
- stdio: `matlab-mcp` (default, for single-agent use)
- Streamable HTTP: `matlab-mcp --transport streamablehttp` or set in config.yaml
- With custom config: `matlab-mcp --config path\to\config.yaml`
- Verify it works: show expected log output lines (the startup banner from server.py lines 782-825).

**First MATLAB tool call section**:
- Show a curl command to test the /mcp endpoint (or note that agents handle this automatically).
- Show expected success: server logs "Job ... completed" and returns result.

**Troubleshooting section** (5-6 common issues):
- "MATLAB engine not found": Check MATLAB Engine API installation, `python -c "import matlab.engine"`
- "Address already in use": Another process on port 8765, change port in config
- "Windows Firewall popup": You set host to 0.0.0.0 without a firewall rule, switch back to 127.0.0.1
- "Permission denied on temp dir": Ensure `execution.temp_dir` points to a user-writable location (default `./temp` is fine)
- "Token rejected (401)": Verify MATLAB_MCP_AUTH_TOKEN env var is set in the same shell session

**HITL note**: Brief mention that human-in-the-loop approval gates can be enabled in config.yaml for protected MATLAB functions. Link to config.yaml comments for the `hitl:` section.
